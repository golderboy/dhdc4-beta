# คู่มือติดตั้ง DHDC4 สำหรับผู้ดูแลระบบ

เอกสารชุดนี้ใช้กับแอปพลิเคชันและชุดติดตั้งฐานข้อมูล DHDC4 รุ่น `v4.0.2` ฐานข้อมูลในชุดติดตั้งเป็นฐานตั้งต้นที่ยังไม่มีข้อมูล 43 แฟ้ม ข้อมูลนำเข้าค้าง รายงาน HDC Exchange ค้าง หรือบัญชีผู้ใช้งานแอปพลิเคชัน

เลือกอ่านคู่มือให้ตรงกับเครื่องที่จะติดตั้ง ไม่ควรนำคำสั่งของสองระบบปฏิบัติการมาปนกัน

- Windows PC หรือ Windows Server ที่ใช้ Apache/XAMPP: เปิด `INSTALL-WINDOWS-TH.md`
- Linux ที่ใช้ Apache: เปิด `INSTALL-LINUX-TH.md`

สำหรับ Linux ห้ามสมมติว่า MariaDB ใช้พอร์ต `3306` ให้รัน `install-linux.sh --check-connection` เพื่อเชื่อมต่อผ่าน Unix socket และอ่านพอร์ตจริงจาก `@@port` ก่อนติดตั้ง ตัวติดตั้งจะหยุดหากพอร์ตที่กำหนดไว้ไม่ตรงกับ MariaDB ที่กำลังทำงาน

## สิ่งที่ตัวติดตั้งฐานข้อมูลทำให้

ตัวติดตั้งจะตรวจ checksum ของทุกไฟล์ นำเข้า SQL ตามลำดับ สร้างบัญชี MariaDB `'dhdc4'@'localhost'` ให้สิทธิเฉพาะฐาน `dhdc4` และตรวจโครงสร้างกับข้อมูลตั้งต้นหลังติดตั้ง รหัสผ่านของบัญชีฐานข้อมูลจะถามบนหน้าจอแบบซ่อนข้อความและไม่ถูกเก็บไว้ในแพ็กเกจ

การติดตั้งสำเร็จต้องมีข้อความขึ้นต้นดังนี้:

```text
DHDC4_VERIFY PASS
```

หากไม่พบคำว่า `PASS` ห้ามเปิด Apache ให้ผู้ใช้งาน และให้เปิดไฟล์ log ในโฟลเดอร์ `install-logs` เพื่อตรวจข้อผิดพลาด

## บัญชี Admin ตั้งต้นของแอปพลิเคชัน

หลังติดตั้งฐานข้อมูลและตั้งค่า Yii2 แล้ว ให้สร้าง Admin คนแรกด้วยสคริปต์ `tools/bootstrap-admin.php` ตามคู่มือของระบบปฏิบัติการ

- Username: `admin`
- Password: `P@ssw0rd`
- Email: ผู้ติดตั้งต้องกรอกอีเมลจริงของผู้ดูแลระบบ

รหัสผ่านนี้เป็นรหัสผ่านชั่วคราวที่ทราบกันทั่วไป ต้องเปลี่ยนทันทีหลังเข้าสู่ระบบครั้งแรก และต้องเปลี่ยนก่อนเปิดระบบให้เครื่องอื่นเข้าถึง

## ข้อควรจำ

- บัญชี MariaDB และบัญชี Admin ของแอปเป็นคนละบัญชีกัน
- ห้ามวางไฟล์ SQL, backup, `.env`, log หรือรหัสผ่านไว้ใน `frontend/web` และ `backend/web`
- ห้ามตั้ง Apache `DocumentRoot` ไปที่โฟลเดอร์รากของโปรเจกต์
- `event_scheduler` จะถูกปิดไว้หลังติดตั้ง ให้เปิดเมื่อทดสอบ timezone และ `event_dhdc` แล้วเท่านั้น
- ก่อนติดตั้งทับฐานเดิม ต้องมี backup และไฟล์ SHA-256 ที่ตรวจสอบได้
